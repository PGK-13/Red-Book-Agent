# Module B (Knowledge) 前端改动清单

> 基于苏格拉底式对话梳理，前端需要配合实现的接口对接和功能变更。

---

## 一、需新增/改动的接口对接

### 1. 文档上传流程（变动大）

**现状**：前端有拖拽上传 UI，但没有实际上传逻辑和状态轮询。

**需要改的地方**：
- 拖拽区域绑定文件选择 + 上传 API
- 上传后显示 `pending` 状态
- 增加轮询机制，查询 `GET /api/v1/knowledge/documents/{id}` 获取 `status`
- 状态变化：`pending` → `processing` → `indexed` 或 `failed`
- 失败时显示错误原因，支持重试

**对应接口**：
- `POST /api/v1/knowledge/documents` — 上传文件
- `GET /api/v1/knowledge/documents/{id}` — 轮询状态

---

### 2. 爆款文案导入流程（新增功能）

**现状**：没有"从已发布笔记导入"的入口。

**需要改的地方**：
- 在爆款文案列表页增加"从笔记导入"按钮
- 点击后弹出选择已发布笔记的弹窗（需要账号下的笔记列表）
- 确认后调用 `POST /api/v1/knowledge/viral-copies/import-from-notes`
- 导入成功后刷新列表

**对应接口**：
- `POST /api/v1/knowledge/viral-copies/import-from-notes` — 从笔记导入

---

### 3. 统计卡片数据（改动小）

**现状**：文档数量、待处理文件、索引条目等是 mock 静态数据。

**需要改的地方**：
- 页面加载时调用 `GET /api/v1/knowledge/stats`
- 用返回的真实数据替换静态值

**对应接口**：
- `GET /api/v1/knowledge/stats` → 填充 4 个 StatCard

---

### 4. 行业爆款笔记详情（新增页面）

**现状**：没有行业笔记详情页入口。

**需要改的地方**：
- 点击行业笔记列表中的某条，打开详情弹窗或详情页
- 展示字段：标题、正文摘要、标签、封面风格、互动数据、发布时间、粉丝量级
- 增加"重新抓取"按钮

**对应接口**：
- `GET /api/v1/knowledge/industry-notes/{id}` — 笔记详情
- `POST /api/v1/knowledge/industry-notes/{id}/re-crawl` — 重新抓取

---

### 5. 检索设置面板（新增功能）

**现状**：UI 上有"混合检索开关"和"向量权重"显示区域，但没有实际配置功能。

**需要改的地方**：
- 点击设置入口，调用 `GET /api/v1/knowledge/settings` 获取当前配置
- 编辑后调用 `PUT /api/v1/knowledge/settings` 保存
- 展示 vector_weight 和 hybrid_enabled 两个字段

**对应接口**：
- `GET /api/v1/knowledge/settings`
- `PUT /api/v1/knowledge/settings`

---

## 二、不需要前端改动的部分

| 问题 | 原因 |
|------|------|
| 文档格式扩展（Excel/PPT） | 后端留好 enum 值后，前端下拉框加个选项即可 |
| 多商家数据隔离 | 后端数据隔离，前端无感知 |
| 行业数据参与内容生成 | 属于内部触发逻辑，前端不涉及 |

## 三、待确认事项（影响前端实现）

### 1. 行业数据触发时机

**问题**：B6 趋势分析是用户手动触发，还是内容生成时自动引用行业爆款数据？

- 如果是**手动触发**：前端需要"趋势分析"按钮和结果展示
- 如果是**自动参与生成**：前端可能在内容生成结果页展示参考了哪些行业爆款

**建议**：明确后再定前端交互。

---

### 2. 对话上下文入口

**问题**：B4 对话上下文是对内的 API（Agent 调用），前端是否有独立页面展示用户的对话历史？

- 如果**有独立页面**：需要展示每个小红书用户的对话记录、偏好标签
- 如果**只在对话中体现**：前端无需单独开发

**建议**：确认 Interaction 模块是否有单独的对话历史页面。

## 四、优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 文档上传 + 状态轮询 | 核心功能，当前完全不可用 |
| P0 | 统计卡片数据对接 | 影响页面核心指标展示 |
| P1 | 检索设置面板 | 有 UI 但无功能 |
| P1 | 行业笔记详情 + 重新抓取 | 有列表无详情 |
| P2 | 爆款文案从笔记导入 | 增强功能，可后续迭代 |
| P3 | 趋势分析页面 | 取决于触发时机确认 |
