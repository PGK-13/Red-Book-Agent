# 模块 A：账号集成与基础配置 (Account & Integration)

**核心价值**：解决"怎么进入小红书"的问题，是所有自动化操作的前提。

---

## 需求 A1：账号接入管理

**用户故事：** 作为商家，我希望系统支持多种接入方式并能统一管理多个小红书账号的授权信息，以便灵活适配不同账号类型并确保账号安全。

### 验收标准

1. THE 营销 Agent SHALL 支持以下三种账号接入方式：小红书官方开放平台 API（适用于专业号）、第三方 RPA 工具和浏览器自动化脚本，商家可按账号类型选择对应接入方式。
2. THE 营销 Agent SHALL 支持商家在同一后台管理多个小红书账号，账号数量上限由商家套餐配置决定。
3. WHEN 商家通过官方开放平台 API 接入账号，THE 营销 Agent SHALL 使用 OAuth 2.0 协议完成授权，并将访问令牌加密存储。
4. WHEN 账号的 Cookie 距离过期时间不足 24 小时，THE 营销 Agent SHALL 向商家发送 Cookie 刷新提醒通知。
5. IF Cookie 已过期且商家未完成刷新，THEN THE 营销 Agent SHALL 暂停该账号的所有自动化操作，并将账号状态标记为"授权失效"。
6. THE 营销 Agent SHALL 支持商家为每个账号配置独立的语气人设（如：专业、活泼、种草风），人设配置作为 LLM System Prompt 的一部分注入内容生成流程。
7. WHEN 商家选择浏览器自动化方式接入账号，THE 营销 Agent SHALL 提供扫码登录功能：系统通过 Playwright 打开小红书登录页并截取二维码图片返回给前端展示，商家使用小红书 App 扫码确认后，系统自动提取登录后的 Cookie 并加密存储，全程无需商家手动复制粘贴 Cookie。
8. WHEN 扫码登录流程启动后，THE 营销 Agent SHALL 以不超过 3 秒的间隔轮询检测登录状态，整个扫码登录会话的有效期为 5 分钟，超时后自动关闭浏览器上下文并返回超时状态。

## 需求 A2：代理与环境隔离

**用户故事：** 作为商家，我希望系统能为每个账号隔离网络环境，以便降低多账号操作被平台关联风控的概率。

### 验收标准

1. THE 营销 Agent SHALL 支持为每个账号配置独立的代理 IP，同一账号的所有自动化操作请求 SHALL 始终通过该账号绑定的代理 IP 发出。
2. IF 某账号未配置代理 IP，THEN THE 营销 Agent SHALL 在执行自动化操作前向商家发出警告提示，说明存在 IP 混用风险。
3. THE 营销 Agent SHALL 支持为每个账号配置独立的浏览器环境（User-Agent、屏幕分辨率、时区），模拟真实设备指纹，不同账号之间的设备指纹参数不得重复。

---

## 需求 A3：账号画像同步

**用户故事：** 作为商家，我希望系统能自动抓取并维护账号的基础信息，以便 LLM 生成内容时能匹配账号人设风格。

### 验收标准

1. WHEN 商家完成账号接入，THE 营销 Agent SHALL 自动抓取该账号的昵称、简介、标签、粉丝数和最近 20 篇历史笔记的文案，生成账号画像并存入数据库。
2. THE 营销 Agent SHALL 每 24 小时自动刷新一次账号画像数据，确保粉丝数等动态数据保持最新。
3. WHEN 内容生成引擎生成笔记文案，THE 内容生成引擎 SHALL 将目标账号的账号画像作为上下文输入 LLM，使生成内容与账号人设保持一致。

---

## 需求 A4：账号状态监控

**用户故事：** 作为商家，我希望系统能实时检测账号的在线状态和健康度，以便在账号出现异常时第一时间收到告警。

### 验收标准

1. THE 营销 Agent SHALL 每 10 分钟对所有已接入账号执行一次状态探测，检测账号是否处于正常可操作状态。
2. WHEN 状态探测发现账号异常（包括：被封禁、被限流、Cookie 失效、API 授权撤销），THE 营销 Agent SHALL 立即停止该账号的所有自动化操作并向商家发送告警通知。
3. IF 外部接口返回账号异常或封禁状态码，THEN THE 营销 Agent SHALL 立即停止该账号的所有自动化操作并记录包含时间戳和错误码的异常日志。
4. THE 营销 Agent SHALL 在后台提供账号健康度仪表盘，展示每个账号的当前状态、最近操作时间和异常历史记录。
