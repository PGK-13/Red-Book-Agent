# 小红书营销自动化 Agent — 需求文档索引

## 模块划分

| 模块 | 目录 | 核心价值 |
|------|------|----------|
| A. 账号集成与基础配置 | `module-a-account/` | 解决"怎么进入小红书"的问题，是所有自动化操作的前提 |
| B. RAG 知识库与记忆 | `module-b-knowledge/` | 解决"Agent 懂不懂行"的问题，提供事实依据和品牌风格 |
| C. 内容创作引擎 | `module-c-content/` | 解决"发什么"的问题，实现文案和图片的自动化生产 |
| D. 互动与意图路由 | `module-d-interaction/` | 解决"怎么回"的问题，是 Agent 的核心大脑 |
| E. 风控与合规保障 | `module-e-risk/` | 解决"安不安全"的问题，防止封号和违规 |
| F. 数据看板与人机协作 | `module-f-analytics/` | 解决"好不好用"的问题，让商家能监控并干预 Agent |

## 模块依赖关系

```
[A 账号模块] ──握手建立连接──▶ [C 内容引擎] ──发布内容──▶ 小红书平台
                                                              │
                                                              ▼
[B 知识库模块] ◀──检索答案── [D 互动模块] ◀──监听回传──────┘
      │
      └──动态权重反哺──▶ [F 数据看板]
                              ▲
[E 风控模块] ──过滤所有进出信息──┘（横切所有模块）
```

## 术语表

- **营销 Agent**：本系统整体，负责协调各子模块完成营销自动化任务的智能体。
- **账号画像**：系统从小红书账号抓取的基础信息集合（昵称、简介、标签、粉丝数、历史笔记风格），供 LLM 生成内容时匹配账号人设。
- **LLM**：大语言模型，本系统默认使用 GPT-4o，接口层设计为可替换。
- **Function Calling**：LLM 的结构化工具调用能力，用于意图分类和动作路由。
- **RAG 知识库**：基于向量数据库存储品牌产品手册和历史爆款文案，供检索增强生成使用的模块。
- **爆款文案库**：历史上在小红书平台获得高互动数据的优质文案集合。
- **内容生成引擎**：负责生成符合小红书平台风格的图文/视频内容的模块。
- **意图识别路由**：通过 Function Calling 对用户评论或私信进行意图分类并路由到对应处理逻辑的模块。
- **评论监测器**：持续监控指定笔记评论区新增评论的模块。
- **私信转化器**：根据意图识别结果自动触发私信回复以完成销售导流的模块。
- **发布调度器**：负责管理笔记定时发布任务、维护发布队列并在指定时间触发发布动作的模块。
- **封面模板引擎**：将 LLM 生成的标题文字自动套入商家预设封面模板并输出首图的模块。
- **OCR 识别器**：对评论区用户发送的图片进行光学字符识别，提取图片中文字内容供意图分类使用的模块。
- **风控过滤器**：在内容发布和回复前进行敏感词检测与平台规则合规校验的模块。
- **竞品避嫌过滤器**：检测评论或私信中是否包含竞品关键词，命中时阻止自动回复并转入人工审核的模块。
- **HITL 审核工作台**：供运营人员对 Agent 准备好的回复执行审核操作（通过、修改后通过、拒绝）的后台界面。
- **人工接管机制**：当特定触发条件满足时，系统暂停 Agent 自动回复并将会话移交人工处理的流程。
- **用户**：小红书平台上与商家笔记产生互动的普通用户。
- **商家**：使用本系统的小红书品牌方或运营人员。
