# 模块 F：数据看板与人机协作 (Analytics & HITL)

**核心价值**：解决"好不好用"的问题，让商家能监控 Agent 运行状态、干预关键决策并持续优化策略。

---

## 需求 F1：转化漏斗分析

**用户故事：** 作为商家，我希望能查看从内容曝光到最终成交的完整链路数据，以便评估 ROI 并定位转化瓶颈。

### 验收标准

1. THE 营销 Agent SHALL 提供以下转化漏斗核心指标的每日统计报表：笔记曝光量、评论互动数、私信触发数、私信转化率（触发购买意向 / 总私信数）、最终成交数（需商家手动录入或对接电商平台）。
2. THE 营销 Agent SHALL 记录每次 LLM 调用的输入、输出、耗时和 token 消耗，日志保留周期不少于 30 天。
3. THE 营销 Agent SHALL 提供 RESTful API 接口，供商家后台系统查询实时运行状态和历史统计数据。
4. THE 营销 Agent SHALL 支持按账号、按时间范围、按内容类型对统计数据进行筛选和导出（CSV 格式）。

---

## 需求 F2：人工干预工作台（HITL）

**用户故事：** 作为运营人员，我希望能在统一的审核工作台对 Agent 的回复进行人工干预，以便在保持自动化效率的同时确保关键场景的回复质量。

### 验收标准

1. THE HITL 审核工作台 SHALL 提供统一的后台界面，展示 Agent 准备好但尚未发送的回复内容，供运营人员执行"通过"、"修改后通过"或"拒绝"操作。
2. THE HITL 审核工作台 SHALL 支持运营人员对多条待审核回复执行批量审核操作。
3. WHEN 运营人员选择"修改后通过"，THE HITL 审核工作台 SHALL 提供内联编辑功能，允许运营人员直接修改回复内容后提交发送。
4. THE HITL 审核工作台 SHALL 展示每条待审核回复的意图分类结果、置信度分数和触发来源（评论 / 私信），辅助运营人员快速判断。
5. THE 私信转化器 SHALL 记录每次私信交互的意图分类结果、回复内容和用户响应状态，所有记录在 HITL 工作台中可查询。

## 需求 F3：异常告警中心

**用户故事：** 作为商家，我希望系统能主动推送各类异常告警，以便在账号风险、API 异常和系统错误发生时第一时间响应。

### 验收标准

1. WHEN 系统任意核心模块发生错误，THE 营销 Agent SHALL 在 1 分钟内通过配置的告警渠道（Webhook 或邮件）发送包含错误类型、模块名称和时间戳的告警消息。
2. THE 营销 Agent SHALL 监控 LLM API 的余额和调用配额，WHEN 余额低于商家设定的预警阈值，THE 营销 Agent SHALL 向商家发送余额不足提醒。
3. WHEN 营销 Agent 检测到单账号在 1 小时内的操作频率超过阈值，THE 营销 Agent SHALL 自动暂停该账号的自动化操作并向商家发送告警通知。
4. THE 营销 Agent SHALL 提供告警历史记录页面，展示所有历史告警的类型、时间、处理状态和处理人。
5. THE 营销 Agent SHALL 支持商家配置告警静默规则，对指定类型的告警在特定时间段内不重复推送。
