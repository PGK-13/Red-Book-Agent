# 部署安全规范 — 禁止直接执行部署命令

## 绝对规则

禁止建议、展示、生成或执行任何将变更部署、推送或应用到远程环境（测试环境、生产环境或任何共享基础设施）的命令。无例外。

## 禁止执行的命令（非完整列表）

- `docker push` / `docker-compose up` 针对远程环境
- 任何携带生产环境连接串、IP 或域名的 `psql` / `pg_dump` / `pg_restore`
- 直接 SSH 到生产服务器执行变更
- 直接修改生产环境的 `.env` 或配置文件
- `kubectl apply` / `helm upgrade` 针对生产集群（如未来引入 K8s）
- 任何包含生产数据库 URL 或 API Key 的命令

## 正确做法

- 所有部署通过 GitHub Actions CI/CD 流水线完成（合并到 `main` 触发测试环境，打 `v*` tag 触发生产环境）
- 需要部署时，告知用户推送/合并代码，由流水线自动处理
- 本地开发环境（`docker-compose up` 针对本地）完全允许
- 编写迁移 SQL 文件没问题，针对远程执行不行

## 原因

生产环境的任何直接变更都可能导致数据丢失或服务中断。所有远程变更必须经过代码审查和自动化流水线。
